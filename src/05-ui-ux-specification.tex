\section{Screen States and UX Rules}

\begin{longtable}{>{\bfseries}p{0.24\textwidth}p{0.66\textwidth}}
\toprule
State category & Required UX behavior \\
\midrule
Loading state & Show deterministic loading indicators and preserve context for long-running tasks. \\
Empty state & Explain missing data state and provide a primary next action. \\
Validation state & Inline field validation with clear error messaging and corrective guidance. \\
Permission denied state & Show role or license reason and escalation path. \\
Blocked compliance state & Prevent restricted actions when operator license or insurance is invalid. \\
Long-running task state & Surface task ID, current stage, elapsed time, retry option, and support route. \\
Offline or failure state & Preserve user input and provide retry without data loss. \\
Audit-required state & Require reason capture for overrides and stack switching before submit. \\
\bottomrule
\end{longtable}

\section{Handoff Requirements for UI/UX Team}

\begin{enumerate}[leftmargin=*]
    \item Provide low-fidelity flow maps for each workflow with success, alternate, and failure paths.
    \item Deliver high-fidelity desktop and mobile designs for every required screen ID.
    \item Define component library usage for forms, tables, timelines, status badges, alerts, drawers, and modals.
    \item Specify interaction behavior for stack switching, license gating, and Spark task status monitoring.
    \item Provide content guidelines for labels, errors, confirmations, and compliance messaging.
    \item Deliver annotated prototypes with API data contracts expected by each screen.
    \item Attach accessibility checklist covering keyboard flow, contrast, focus order, and screen-reader labels.
\end{enumerate}

\section{Acceptance Criteria for UX Readiness}

\begin{itemize}[leftmargin=*]
    \item Every workflow in this document is represented by a complete clickable prototype.
    \item Every screen includes loading, empty, validation, and failure states.
    \item Compliance and licensing restrictions are reflected in user journeys and UI controls.
    \item Frame lineage and stack switching workflows are represented end to end.
    \item Spark task visibility is represented for operations users with clear status and escalation paths.
\end{itemize}
